# 进度报告模板

本文件包含 Feature-Agent 使用的进度报告模板。

## 1. 任务完成进度汇报模板

```text
[EXECUTE-PROGRESS] 任务完成:
- 任务ID: <task_id>
- 任务名称: <task_name>
- 任务类型: <task_type>
- Wave: <wave_id>
- 状态: completed
- 验证结果: <通过/失败>
- 执行时间: <开始时间> - <结束时间> (<耗时>)

任务摘要:
- 修改文件数: <N>
- 新增测试数: <M>
- 验收标准满足: <全部/部分>

当前 Wave 进度:
- Wave: <wave_id>
- Wave 名称: <wave_name>
- 已完成: <N> 个任务
- 总任务: <M> 个任务
- 进度: <百分比>
- 剩余任务: <列出>
```

---

## 2. Wave 完成汇报模板

```text
[EXECUTE-WAVE] Wave 完成:
- Wave: <wave_id>
- Wave 名称: <wave_name>
- 开始时间: <时间>
- 完成时间: <时间>
- 总耗时: <时间>
- 完成任务数: <N>
- 总任务数: <M>
- 验证通过率: <百分比>

Wave 任务列表:
- <task_id>: <task_name> - <通过/失败>
- <task_id>: <task_name> - <通过/失败>
- ...

Wave 总结:
- 核心成果: <简要说明>
- 关键实现: <列出关键实现点>
- 测试覆盖: <说明测试情况>
- 遗留问题: <如有问题，列出>

下一步: 进入 <next_wave_id> (开始<next_wave_name>) / 进入总结阶段
```

---

## 3. 执行阶段整体进度汇报模板

```text
[EXECUTE-OVERALL] 执行阶段进度:
- 当前 Wave: <current_wave>
- 已完成 Waves: <list>
- 未完成 Waves: <list>
- 已完成任务数: <N>
- 总任务数: <M>
- 总进度: <百分比>

各 Wave 进度:
- Wave 1 (核心实现): <已完成/进行中/未开始> - <进度百分比>
- Wave 2 (扩展功能): <已完成/进行中/未开始> - <进度百分比>
- Wave 3 (测试验证): <已完成/进行中/未开始> - <进度百分比>
- Wave 4 (文档完善): <已完成/进行中/未开始> - <进度百分比>

风险状态:
- 高风险任务: <已完成/进行中/未开始>
- 验证失败次数: <N>
- 人工介入次数: <M>
- 当前阻塞: <如有阻塞，说明原因>

性能指标:
- 平均任务耗时: <时间>
- 总耗时: <时间>
- 验证通过率: <百分比>

预计剩余时间: <估算>
```
