# 代码检视输出模板

本文件包含 Review Skill 使用的代码检视输出模板。

**重要约定**：所有过程文档存放在 `{kb_dir}/` 目录下，该目录由调用方指定。

---

## 1. 变更范围越界模板

```text
REVIEW_FAIL
reason: OUT_OF_SCOPE
detail: 发现越界修改
severity: high
escalate: true

越界文件:
- <file>: <原因说明>

建议: 回滚越界修改，保持在批准的文件范围内
```

---

## 2. 代码质量违规模板

```text
REVIEW_FAIL
reason: QUALITY_VIOLATION
detail: 代码质量未达标
severity: medium
escalate: false

质量违规:
- <违规项>: <具体问题>
  - 文件: <文件路径>
  - 行号: <行号>
  - 问题: <具体描述>

建议: 在任务边界内改善代码质量
```

---

## 3. 接口兼容性破坏模板

```text
REVIEW_FAIL
reason: INTERFACE_INCOMPATIBLE
detail: 接口兼容性被破坏
severity: high
escalate: true

兼容性问题:
- <接口>: <问题说明>
  - 变更类型: <签名变更/返回类型变更/参数变更/语义变更>
  - 影响范围: <受影响的调用方>

建议: 保持接口向后兼容，或上报用户决策
```

---

## 4. 检视通过输出模板

```text
REVIEW_PASS
task_id: <task_id>
task_name: <task_name>
task_type: <task_type>

检视结果:
✅ 变更范围审计通过 - 所有修改都在批准范围内
✅ 代码质量达标 - 符合项目代码规范
✅ 接口兼容性保持 - 未破坏现有接口兼容性

详细检视信息:
- 检视文件数: <N>
- 检视通过项: <M>
- 检视违规项: 0

检视日志已写入: {kb_dir}/review-log.md
```

---

## 5. 检视未通过输出模板

```text
REVIEW_FAIL
task_id: <task_id>
task_name: <task_name>
task_type: <task_type>

失败原因: <reason>
详情: <detail>
严重程度: <severity> (high/medium/low)

影响范围:
- 受影响的文件: <列出>
- 违规项数: <N>

修复建议:
1. <具体修复步骤1>
2. <具体修复步骤2>
3. <具体修复步骤3>

<具体违规模板内容>
```
