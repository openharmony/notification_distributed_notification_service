# OpenHarmony C/C++ 安全编码检视清单

**来源：华为C/C++安全编码规范 + OpenHarmony安全编程指南**

本文档整合华为和OpenHarmony官方安全编码规范，提供完整的代码检视检查点。

---

## 一、内存安全 (Memory Safety) - P0

### 1.1 内存分配安全

| 检查项 | 检查方式 | 示例 | 规范要求 |
|--------|----------|------|----------|
| **分配前校验大小** | 人工审查 | `if (size == 0 || size > MAX) return;` | 分配前必须校验size != 0且不超过上限 |
| **分配失败检查** | 检查new/malloc后是否判空 | `if (ptr == nullptr) return;` | new/malloc后必须检查返回值 |
| **禁止new[0]** | 检查new unsigned char[0]等 | 不允许 | `new T[0]`行为未定义，禁止使用 |
| **使用安全函数** | grep `malloc\|calloc\|realloc` | 应使用`malloc_s`等安全版本 | OpenHarmony要求使用安全内存函数 |

### 1.2 内存释放安全

| 检查项 | 检查方式 | 示例 | 规范要求 |
|--------|----------|------|----------|
| **new[]配delete[]** | 检查配对 | `new char[n]`必须用`delete[]` | `new[]`必须用`delete[]`，禁止用`delete` |
| **释放后置nullptr** | 人工审查 | `delete ptr; ptr = nullptr;` | 释放后立即置nullptr |
| **禁止重复释放** | 检查释放路径 | 确保不会多次delete同一指针 | 避免UAF和double-free |
| **禁止释放后使用** | 人工审查 | 释放后不应再访问该内存 | Use-After-Free是严重安全漏洞 |

### 1.3 内存拷贝安全

| 检查项 | 检查方式 | 示例 | 规范要求 |
|--------|----------|------|----------|
| **使用memcpy_s** | grep `memcpy\b` | 替换为`memcpy_s(dest, destSize, src, srcLen)` | 必须使用带长度校验的安全版本 |
| **禁止使用strcpy** | grep `strcpy\b` | 替换为`strcpy_s`或`strncpy_s` | strcpy无长度限制，易导致溢出 |
| **禁止使用strcat** | grep `strcat\b` | 替换为`strcat_s` | strcat无长度限制 |
| **禁止使用sprintf** | grep `sprintf\b` | 替换为`snprintf` | sprintf无长度限制 |

## 二、指针安全 (Pointer Safety) - P0

### 2.1 空指针安全

| 检查项 | 检查方式 | 示例 | 规范要求 |
|--------|----------|------|----------|
| **参数指针校验** | 人工审查 | `if (ptr == nullptr) return;` | 所有指针参数使用前必须校验 |
| **返回值校验** | 人工审查 | `if (GetValue() == nullptr) return;` | 函数返回指针必须校验后再使用 |
| **使用nullptr** | grep `\bNULL\b` | `ptr != nullptr` | 禁止使用NULL或0，必须用nullptr |
| **成员指针校验** | 人工审查 | 访问成员指针前校验 | 类成员指针使用前必须校验 |

### 2.2 悬空指针安全

| 检查项 | 检查方式 | 示例 | 规范要求 |
|--------|----------|------|----------|
| **禁止返回局部指针** | 人工审查 | 不返回局部变量的地址 | 返回局部变量指针会导致悬空指针 |
| **禁止返回栈对象指针** | 人工审查 | 不返回栈上对象的地址 | 栈对象销毁后指针失效 |
| **智能指针生命周期** | 人工审查 | 注意shared_ptr循环引用 | 使用weak_ptr避免循环引用 |

### 2.3 指针运算安全

| 检查项 | 检查方式 | 示例 | 规范要求 |
|--------|----------|------|----------|
| **指针偏移校验** | 人工审查 | `ptr + offset`前校验边界 | 指针偏移不应超出缓冲区范围 |
| **禁止强制转换指针** | 检查C风格转换 | `(int*)ptr` → `reinterpret_cast<int*>(ptr)` | 使用C++类型转换而非C风格 |

## 三、缓冲区安全 (Buffer Safety) - P0

### 3.1 数组访问安全

| 检查项 | 检查方式 | 示例 | 规范要求 |
|--------|----------|------|----------|
| **索引边界校验** | 人工审查 | `if (index >= size) return;` | 数组索引必须校验是否在有效范围 |
| **禁止差一错误** | 人工审查 | `index > size` vs `index >= size` | 注意边界是>=而非> |
| **多维数组校验** | 人工审查 | 校验所有维度的索引 | 每个维度都要校验 |

### 3.2 缓冲区读写安全

| 检查项 | 检查方式 | 示例 | 规范要求 |
|--------|----------|------|----------|
| **写入前校验空间** | 人工审查 | `if (offset + len > bufSize) return;` | 写入前检查剩余空间是否足够 |
| **读取前校验长度** | 人工审查 | `if (offset + len > dataSize) return;` | 读取前检查数据长度是否足够 |
| **禁止依赖'\0'终止** | 人工审查 | 不依赖字符串终止符确定边界 | 使用指定长度而非依赖'\0' |

### 3.3 字符串安全

| 检查项 | 检查方式 | 示例 | 规范要求 |
|--------|----------|------|----------|
| **构造string指定长度** | 人工审查 | `std::string(buf, len)`而非`std::string(buf)` | 使用指定长度构造，不依赖'\0' |
| **禁止空指针创建string** | 人工审查 | `if (ptr == nullptr) return;` | 空指针传给std::string构造函数会崩溃 |
| **禁止从空指针创建string** | 人工审查 | 检查const char*是否为nullptr | 创建string前必须校验指针非空 |
| **禁止越界string访问** | 人工审查 | `str[i]`前检查i < str.size() | string访问也要校验边界 |

## 四、整数安全 (Integer Safety) - P0

### 4.1 整数溢出

| 检查项 | 检查方式 | 示例 | 规范要求 |
|--------|----------|------|----------|
| **加法溢出检查** | 人工审查 | `if (a > MAX - b) return;` | 加法前检查是否会溢出 |
| **乘法溢出检查** | 人工审查 | `if (a > MAX / b) return;` | 乘法前检查是否会溢出 |
| **使用减法而非加法** | 人工审查 | `total - used > need`而非`used + need > total` | 用减法避免无符号回绕 |
| **有符号整数运算** | 人工审查 | 注意有符号数可能溢出 | 有符号溢出是未定义行为 |

### 4.2 除零安全

| 检查项 | 检查方式 | 示例 | 规范要求 |
|--------|----------|------|----------|
| **除法前检查** | 人工审查 | `if (divisor == 0) return;` | 除法和模运算前必须检查除数不为0 |
| **取模前检查** | 人工审查 | `if (mod == 0) return;` | 取模前检查模数不为0 |
| **隐式除法检查** | 人工审查 | 注意avg = total / count等 | 检查所有可能的除法操作 |

### 4.3 整数类型安全

| 检查项 | 检查方式 | 示例 | 规范要求 |
|--------|----------|------|----------|
| **禁止long类型** | grep `\blong\b` | 使用`int64_t`替代`long` | long在32/64位平台长度不同 |
| **禁止无符号到有符号转换** | 人工审查 | 避免int32_t = uint32_t | 可能导致数值变化 |
| **使用固定宽度类型** | 人工审查 | `int32_t, uint32_t, int64_t` | 使用stdint.h定义的类型 |

## 检视示例

### Bad示例（需修复）

```cpp
// P0-01: 分配前未校验大小
void Foo(uint32_t size) {
    char* buf = new char[size];  // 未校验size == 0
}

// P0-10: 使用NULL
if (ptr != NULL) { }  // 应改为 nullptr

// P0-11: 参数指针未校验
void Process(const char* data) {
    printf("%s", data);  // 未校验data是否为nullptr
}

// P0-19: 依赖'\0'终止符
std::string str(reinterpret_cast<char*>(buffer));  // 未指定长度

// P0-24: 除法未检查
int avg = total / count;  // count可能为0

// P0-07: 使用memcpy
memcpy(dest, src, len);  // 应改为memcpy_s
```

### Good示例（符合规范）

```cpp
// Good: 分配前校验大小
void Foo(uint32_t size) {
    if (size == 0 || size > MAX_SIZE) {
        ANS_LOGW("Invalid size %{public}u", size);
        return;
    }
    char* buf = new char[size];
    if (buf == nullptr) {
        ANS_LOGE("Allocation failed");
        return;
    }
    // ...
    delete[] buf;
    buf = nullptr;
}

// Good: 使用nullptr + 校验
void Process(const char* data, uint32_t len) {
    if (data == nullptr || len == 0) {
        return;
    }
    // 使用指定长度
    std::string str(data, len);
}

// Good: 使用memcpy_s
errno_t err = memcpy_s(dest, destSize, src, srcLen);
if (err != EOK) {
    ANS_LOGE("memcpy failed");
    return;
}

// Good: 除法前检查
if (count == 0) {
    return;
}
int avg = total / count;
```
