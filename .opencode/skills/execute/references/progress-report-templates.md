# 进度报告模板

本文件包含工作流编排方使用的进度报告模板。

## 1. 任务完成进度汇报模板

```text
[EXECUTE-PROGRESS] 任务完成:
- 任务ID: <task_id>
- 任务名称: <task_name>
- 任务类型: <task_type>
- 状态: completed
- 检视结果: <通过/未通过>
- 编译验证: <通过/失败/跳过>
- 执行时间: <开始时间> - <结束时间> (<耗时>)

任务摘要:
- 修改文件数: <N>
- 新增测试数: <M>

当前进度:
- 已完成: <N> 个任务
- 总任务: <M> 个任务
- 进度: <百分比>
- 剩余任务: <列出>
```

---

## 2. 执行阶段整体进度汇报模板

```text
[EXECUTE-OVERALL] 执行阶段进度:
- 已完成任务数: <N>
- 总任务数: <M>
- 总进度: <百分比>

按任务类型进度:
- 核心实现: <已完成/进行中/未开始> - <进度百分比>
- 扩展功能: <已完成/进行中/未开始> - <进度百分比>
- 测试验证: <已完成/进行中/未开始> - <进度百分比>
- 文档完善: <已完成/进行中/未开始> - <进度百分比>

风险状态:
- 高风险任务: <已完成/进行中/未开始>
- 检视未通过次数: <N>
- 编译失败次数: <M>
- 人工介入次数: <K>
- 当前阻塞: <如有阻塞，说明原因>

性能指标:
- 平均任务耗时: <时间>
- 总耗时: <时间>
- 编译通过率: <百分比>

预计剩余时间: <估算>
```
