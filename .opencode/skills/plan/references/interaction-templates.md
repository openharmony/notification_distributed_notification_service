
### 阶段 5: 整体验证与审批

```text

- 总任务数: <N>

- 测试计划: <简要说明>
- 文档计划: <简要说明>

功能覆盖审计:
- 总功能点: <N>个
- 已覆盖: <M>个 (覆盖率 <百分比>%)
- 未覆盖需补充: <X>个
- 刻意排除: <Y>个

- 对齐开发设计项: <M>个
- 新增项 [NEW]: <N>个

文档位置: {kb_dir}/plan.md

请明确回复"批准"或"需要修改"后,才能进入实现阶段。
```

## Plan 确认门模板

```text
[PLAN-APPROVAL] 任务计划已生成,摘要如下:

总任务数: <N>
按类型分布:
- 核心实现: <任务数>个
- 扩展功能: <任务数>个
- 测试验证: <任务数>个
- 文档完善: <任务数>个

高风险任务: <列出>

功能覆盖审计:
- 总功能点: <N>个
- 已覆盖: <M>个 (覆盖率 <百分比>%)
- 刻意排除 [EXCLUDED]: <Y>个 (原因: <列出>)

溯源追踪:
- 对齐开发设计项: <M>个
- 新增项 [NEW]: <N>个 (原因: <列出>)

文档位置: {kb_dir}/plan.md

请确认任务计划:
- **批准**: 进入 Execute 阶段，开始任务执行
- **需要修改**: Plan 继续迭代，调整任务计划
- **拒绝**: 回退到 Dev-Design 阶段，重新讨论设计方案

请明确回复: [批准/需要修改/拒绝]
```
